## 1. 서비스 개요

### 1.1 기본 정보

| 구분 | 내용 |
|------|------|
| **서비스명** | 토스 대나무숲 |
| **슬로건** | "말 못할 이야기, 익명으로 외쳐요" |
| **플랫폼** | 토스 앱 내 미니앱 (Granite/React Native) |
| **대상 사용자** | 토스 앱 사용자 전원 |

### 1.2 MVP 목표

토스 사용자라면 **누구나 1초 만에 진입**해 익명으로 글을 쓰고, **본인이 쓴 글만** 깔끔하게 지울 수 있는 최소 기능 구현.

### 1.3 핵심 가치 제안

- **진입 장벽 제로**: 별도 가입 없이 토스만 있으면 즉시 이용 가능
- **진정한 익명성**: 랜덤 닉네임, 추적 최소화 설계
- **심리적 안전감**: 차분한 파스텔 톤과 익명 대화 분위기

## 2. 핵심 기능 명세

### 2.1 익명 외치기 (Post: Create & Read)

#### 2.1.1 글쓰기

| 항목 | 내용 |
|------|------|
| **입력 형식** | 텍스트 전용 |
| **글자 제한** | 최대 300자 |
| **제외 기능** | 사진/영상 (MVP에서 제외) |

#### 2.1.2 자동 익명화

- 글을 올리는 순간 **서버에서** 랜덤 닉네임 자동 부여
- 형식: `{랜덤 형용사} + {동물}` (예: 졸린 나무늘보, 용감한 사자)
- 사용자는 닉네임을 고르거나 수정할 수 없음 (완전 자동화)

#### 2.1.3 타임라인

- 전체 사용자의 글을 **최신순**으로 노출
- 무한 스크롤 등으로 끊김 없이 로딩

---

### 2.2 댓글 대화 (Comment: Create & Read)

| 항목 | 내용 |
|------|------|
| **진입점** | 게시글 상세 화면 |
| **익명 댓글** | 모든 댓글은 익명으로 작성 |
| **식별 규칙** | 원글 작성자 = "글쓴이", 나머지 = "익명 1", "익명 2"... |

> 대화 맥락 유지를 위해 글쓴이/익명 번호로 구분하되, 실제 사용자 신원은 드러나지 않음.

---

### 2.3 내 글 지우기 (Delete)

| 항목 | 내용 |
|------|------|
| **수정 여부** | ❌ 수정 불가 (Update 미지원) |
| **삭제 범위** | 본인이 쓴 글/댓글만 삭제 가능 |
| **UI** | 본인 글/댓글에만 [삭제] 버튼 노출 |
| **'내 글 목록'** | 의도적으로 미제공 (추적·저장 느낌 최소화) |

---

### 2.4 공감 (Interaction)

| 항목 | 내용 |
|------|------|
| **기능** | 하트 버튼 |
| **의미** | 가벼운 지지/공감 표시 |
| **제한** | 1인 1글당 1하트 (중복 방지) |

## 3. 사용자 경험(UX) 전략

### 3.1 "귀찮음 제로" 계정 시스템

| 구분 | 설계 |
|------|------|
| **가입 절차** | 없음 |
| **식별 방식** | 토스 SDK의 **비식별 고유 ID(Unique ID)** 사용 |
| **동작 방식** | 백엔드에서 해당 ID로 사용자 자동 인식 |
| **권한 체크** | DB의 `user_id`와 현재 접속자의 ID 대조 → 본인 글에만 삭제 버튼 노출 |

### 3.2 디자인 톤앤매너

| 요소 | 방향 |
|------|------|
| **UI 기준** | 토스 앱 기본 폰트·카드 스타일 계승 (이질감 최소화) |
| **색감** | 부드러운 파스텔 톤 배경 |
| **분위기** | 차분하고 안전한 소통 공간 조성 |

> TDS(Toss Design System) 사용 필수 (앱인토스 검수 기준)

## 4. 데이터베이스 설계

### 4.1 테이블 구조

#### Posts (게시글)

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `id` | PK | 게시글 고유 식별자 |
| `user_id` | FK | 토스 SDK 비식별 고유 ID |
| `content` | TEXT | 글 본문 (최대 300자) |
| `nickname` | VARCHAR | 서버 부여 랜덤 닉네임 (형용사+동물) |
| `created_at` | DATETIME | 작성 시간 |

#### Comments (댓글)

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `id` | PK | 댓글 고유 식별자 |
| `post_id` | FK | 해당 게시글 ID |
| `user_id` | FK | 토스 SDK 비식별 고유 ID |
| `content` | TEXT | 댓글 본문 |
| `created_at` | DATETIME | 작성 시간 |

> "글쓴이" vs "익명 N" 판별은 `post.user_id`와 `comment.user_id` 비교로 처리.

#### Hearts (공감)

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| `post_id` | FK | 게시글 ID |
| `user_id` | FK | 토스 SDK 비식별 고유 ID |

> `(post_id, user_id)` 유니크 제약으로 1인 1하트 보장.

## 5. 운영 및 보안

### 5.1 클린 봇 (콘텐츠 필터링)

- **대상**: 글·댓글 등록 시점
- **필터 항목**: 비속어, 금융 사기 유도 키워드 등
- **동작**: 해당 콘텐츠는 등록 차단 또는 사전 경고

### 5.2 신고 시스템

| 항목 | 내용 |
|------|------|
| **UI** | 게시글/댓글에 [신고하기] 버튼 노출 |
| **처리 기준** | 5회 이상 신고 시 해당 글을 타임라인에서 **자동 숨김** |
| **후속** | 필요 시 관리자 검토·영구 삭제 등 정책 확장 가능 |
