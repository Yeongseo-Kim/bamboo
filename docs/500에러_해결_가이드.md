# @granite-js/react-native 500 에러 해결 가이드

## 원인
`@granite-js/react-native` 패키지의 `package.json`이 `main: "./dist/index.js"`를 가리키지만, **실제 dist/index.js 파일이 패키지에 없음**. (패키지 빌드/배포 이슈)

## 적용한 해결 방법

### 1. 패키지 main 필드 수정 (패치)
- `node_modules/@granite-js/react-native/package.json`의 `main`을 `./src/index.ts`로 변경
- `npm install` 후에도 유지되도록 `scripts/patch-granite-rn.js` + `postinstall` 스크립트 추가

### 2. granite.config.ts resolveRequest (이중 안전장치)
- `@granite-js/react-native` 요청 시 `src/index.ts`로 강제 resolve
- 패치가 적용되지 않은 경우에도 동작하도록 유지

## 사용법
1. **Metro 서버 완전 종료** (Ctrl+C)
2. **캐시 삭제 후 재시작**:
   ```bash
   rm -rf /tmp/metro-* node_modules/.cache 2>/dev/null
   npm run dev
   ```
3. Android: `npm run android:connect` 실행
4. 샌드박스에서 `intoss://bamboo-app` 접속

## 참고
- `npm install` 시 postinstall이 자동으로 패치 적용
- 문제 지속 시: `node scripts/patch-granite-rn.js` 수동 실행 후 Metro 재시작
